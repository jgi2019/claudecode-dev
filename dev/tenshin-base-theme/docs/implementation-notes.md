## 技術方針

### 開発環境

- **Claude Code × VS Code**
- **ローカルで作成 → BASE HTML編集Appに貼り付け**
- ビルドツール不使用（Vite等なし）。素のHTML/CSS/JS
- 外部ライブラリ不使用（パララックスもバニラJS）

### ファイル構成

開発中は分割管理：

```
tenshin-base-theme/
├── src/
│   ├── index.html      ← BASEテーマ本体（テンプレートタグ込み）
│   ├── style.css       ← CSS
│   ├── script.js       ← JS
│   └── assets/         ← 画像・ロゴ・フォント
├── mock/
│   └── preview.html    ← BASEタグをダミー値で置換、ローカル確認用
├── docs/
│   ├── base-tag-reference.md
│   ├── implementation-notes.md
│   ├── design-pc.png
│   └── design-sp.png
└── README.md
```

BASE本番への適用時は、HTMLを1ファイルに統合してエディタに貼り付け。CSS/画像は別ファイルとしてBASEのファイルアップローダー経由。

### レスポンシブ

- モバイルファースト設計
- BASE設定「デフォルトのモバイルテーマを使用」を **OFF**
- ブレイクポイント：
  - モバイル：〜768px
  - タブレット：769px〜1024px
  - PC：1025px〜

### HEROパララックス

- **PC版のみ適用、モバイルでは無効化**
- バニラJS（外部ライブラリ不使用）
- 実装：スクロール量に応じてHERO背景のtransform translateY
- アクセシビリティ：`prefers-reduced-motion: reduce` を尊重

```javascript
(function() {
  if (window.innerWidth <= 768) return;
  if (window.matchMedia('(prefers-reduced-motion: reduce)').matches) return;
  
  const bg = document.querySelector('.hero-bg');
  if (!bg) return;
  
  window.addEventListener('scroll', () => {
    const rate = window.scrollY * 0.4;
    bg.style.transform = `translateY(${rate}px)`;
  }, { passive: true });
})();
```

### パフォーマンス目標

- LCP（Largest Contentful Paint）< 2.5秒
- 画像は適切なサイズで配信（WebP対応は後回し）
- HEROの背景画像のみ優先読み込み

## BASE実装上の制約とその対応

### 1ファイル制約

全ページを1つのHTMLテーマファイルで条件分岐管理。開発中はsrc/配下で分割、本番投入時に統合。

### 必須タグ（非表示・改変禁止）

フッターに必ず含める：
- `{ContactPageURL}` お問い合わせ
- `{PrivacyPageURL}` プライバシーポリシー
- `{LawPageURL}` 特定商取引法

### 画像アップロード

BASE HTML編集AppのCSSアップロード機能経由。独自URLタグで参照。

### JS実行タイミング

BASE環境でスクリプトタグがどう評価されるか、最初の動作確認時に要検証。

## 商品データについて

### 初期商品

**豚まん1商品のみ**でスタート。

### LPへの影響

- 「自宅で愉しむ」セクションの商品カードを3点並びにする計画だったが、現時点では **豚まん1商品の複数アングル** として表現する
- `{block:Items}` ループは最大24件まで表示可能だが、初期は1件しか出ない
- **カスタム商品画像3点** を `{ItemImage1URL-500}` `{ItemImage2URL-500}` `{ItemImage3URL-500}` で取得する設計が自然
  - ただし「2〜20は商品ページのみ使用可」の制約があるため、LP上では使えない
  - → **代替案**：LP上のコラージュは静的なテーマ画像として配置、`{block:Items}` ループは「オンラインショップへ」ボタンのリンク先（商品一覧）で機能させる

### 商品が増えた場合

将来的に商品が増えたら、LP下部に `{block:Items}` ループで最新3〜6商品を自動表示する設計に拡張可能。

## アクセシビリティ

- 画像には必ず `alt` 属性
- フォーカスリングを消さない
- カラーコントラスト比：本文 4.5:1 以上、大きい文字 3:1 以上
- `prefers-reduced-motion` への配慮（パララックス無効化）
- 言語属性 `lang="ja"` 必須
